Customize your Zetty Documentation Hub to fit your needs and branding.

## Base Configuration

The main configuration is handled through Vite's configuration system in `vite.config.ts`:

```typescript
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'

export default defineConfig({
  plugins: [react()],
  base: '/zetty-doc-hub/', // GitHub Pages base path
  build: {
    outDir: 'dist',
    sourcemap: true
  }
})
```

## Deployment Configuration

### GitHub Pages

The application is pre-configured for GitHub Pages deployment with the base path `/zetty-doc-hub/`. 

**To deploy to your own repository:**

1. Update the `base` path in `vite.config.ts`:
```typescript
base: '/your-repo-name/',
```

2. Update the workflow file `.github/workflows/deploy.yml` if needed

### Custom Domain

For a custom domain, set the base path to `/`:

```typescript
export default defineConfig({
  base: '/', // For custom domain or root deployment
  // ... rest of config
})
```

## File Discovery Configuration

The file discovery system uses Vite's glob patterns:

```typescript
// In src/utils/fileTree.ts
const docsModules = import.meta.glob('/src/docs/**/*.{md,mdx}', { eager: false });
```

### Supported File Extensions

Currently supported extensions:
- `.md` - Standard Markdown
- `.mdx` - Markdown with JSX

### Adding New Extensions

To support additional file types, update the glob pattern:

```typescript
const docsModules = import.meta.glob('/src/docs/**/*.{md,mdx}', { eager: false });
```

## Site Configuration

### Environment Variables

Customize your documentation hub using environment variables. Create a `.env` file in the root directory:

```bash
# Site Configuration
VITE_SITE_TITLE="My Documentation Hub"
VITE_SITE_DESCRIPTION="A comprehensive documentation hub for my project"
VITE_SITE_AUTHOR="My Team"

# Navigation
VITE_NAVIGATION_SIDEBAR_TITLE="My Docs"

# Branding
VITE_BRANDING_FAVICON="/my-favicon.ico"

# GitHub Integration
VITE_GITHUB_URL="https://github.com/your-username/your-repo"
```

**Available Configuration Options:**

| Variable | Description | Default |
|----------|-------------|---------|
| `VITE_SITE_TITLE` | Main site title (appears in browser tab) | "Zetty Doc Hub" |
| `VITE_SITE_DESCRIPTION` | Site description for meta tags | "A modern documentation hub..." |
| `VITE_SITE_AUTHOR` | Site author for meta information | "Zetty Doc Hub Team" |
| `VITE_NAVIGATION_SIDEBAR_TITLE` | Sidebar header title | "Documentation" |
| `VITE_BRANDING_FAVICON` | Path to favicon | "/favicon.svg" |
| `VITE_GITHUB_URL` | GitHub repository URL | "https://github.com/your-username/your-repo" |

### Default Configuration

The application includes sensible defaults defined in `src/config/siteConfig.ts`. All configuration is now managed through environment variables with fallbacks to these defaults. You can reference the example configuration file:

```bash
cp .env.example .env
# Edit .env with your preferred values
```

### Additional Environment Variables

For complete configuration options, check the `.env.example` file which includes:

- `VITE_NAVIGATION_HIDDEN_DIRECTORIES` - Comma-separated list of directories to hide from navigation
- `VITE_NAVIGATION_MAX_TOC_LEVEL` - Maximum heading level for table of contents (1-6)
- `VITE_FOOTER_TEXT` - Custom footer text
- `VITE_GITHUB_URL` - GitHub repository URL for source code links

## GitHub Integration

Zetty Doc Hub supports linking to GitHub repositories to provide quick access to source code. When configured, a GitHub icon appears in the header that links to your repository.

### Configuration

Set up GitHub integration by adding your repository URL to your `.env` file:

```bash
# GitHub Integration
VITE_GITHUB_URL="https://github.com/your-username/your-repository-name"
```

### Features

- **Repository Link**: Displays a GitHub icon in the header that links to your repository
- **Flexible URLs**: Supports any Git hosting service (GitHub, GitLab, Bitbucket, etc.)
- **Optional Configuration**: GitHub link only appears when URL is configured
- **Direct Linking**: Uses the exact URL you provide for maximum flexibility

## Advanced Configuration

### Custom Markdown Rendering

The markdown renderer can be customized in `MarkdownRenderer.tsx`:

```typescript
import ReactMarkdown from 'react-markdown';
import remarkGfm from 'remark-gfm';
import rehypeHighlight from 'rehype-highlight';

// Add custom plugins
<ReactMarkdown
  remarkPlugins={[remarkGfm, customRemarkPlugin]}
  rehypePlugins={[rehypeHighlight, customRehypePlugin]}
>
  {content}
</ReactMarkdown>
```

### Router Configuration

Customize routing behavior in `App.tsx`:

```typescript
import { BrowserRouter, HashRouter } from 'react-router-dom';

// Use HashRouter for GitHub Pages without custom domain
<HashRouter basename="/zetty-doc-hub">
  <Routes>
    {/* routes */}
  </Routes>
</HashRouter>
```

## Performance Optimization

### Build Optimization

Configure build optimizations in `vite.config.ts`:

```typescript
export default defineConfig({
  build: {
    rollupOptions: {
      output: {
        manualChunks: {
          vendor: ['react', 'react-dom'],
          markdown: ['react-markdown', '@mdx-js/react']
        }
      }
    }
  }
})
```

### Code Splitting

Large markdown files are automatically code-split thanks to Vite's dynamic imports.
