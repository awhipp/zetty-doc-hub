Customize your Zetty Documentation Hub to fit your needs and branding.

## Base Configuration

The main configuration is handled through Vite's configuration system in `vite.config.ts`.

## Deployment Configuration

This application is a single page application (SPA) built with Vite. If your deployment server requires a specific base path, you can set it via environment variables.

### Supported File Extensions

Currently supported extensions:
- `.md` - Standard Markdown
- `.mdx` - Markdown with JSX
- Image files like `.png`, `.jpg`, `.svg`, etc.

### Environment Variables

Customize your documentation hub using environment variables. Create a `.env` file in the root directory:

**Available Configuration Options:**

| Variable | Description | Default |
|----------|-------------|---------|
| `VITE_SITE_TITLE` | Main site title (appears in browser tab) | "Zetty Doc Hub" |
| `VITE_SITE_DESCRIPTION` | Site description for meta tags | "A modern documentation hub..." |
| `VITE_SITE_AUTHOR` | Site author for meta information | "Zetty Doc Hub Team" |
| `VITE_NAVIGATION_SIDEBAR_TITLE` | Sidebar header title | "Documentation" |
| `VITE_BRANDING_FAVICON` | Path to favicon | "/favicon.svg" |
| `VITE_GITHUB_URL` | GitHub repository URL | "https://github.com/your-username/your-repo" |
| `VITE_NAVIGATION_HIDDEN_DIRECTORIES` | Comma-separated list of directories to hide from navigation | "" |
| `VITE_NAVIGATION_MAX_TOC_LEVEL` | Maximum heading level for table of contents (1-6) | 2 |
| `VITE_FOOTER_TEXT` | Custom footer text | "Built with Zetty Doc Hub" |

### Default Configuration

The application includes sensible defaults defined in `src/config/siteConfig.ts`. All configuration is now managed through environment variables with fallbacks to these defaults. You can reference the example configuration file:

```bash
cp .env.example .env
# Edit .env with your preferred values
```
